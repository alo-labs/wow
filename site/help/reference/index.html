CLI Commands

All WOW! commands are invoked via WP-CLI from your WordPress root directory.

wp package install alo-labs/wow   # Install WOW!
/wow                               # Run full optimization pipeline
/wow --dry-run                     # Audit only — no changes made
/wow --domain=example.com          # Target a specific domain
/wow --rollback                    # Revert to last backup
/wow --report                      # Display last run report
/wow --version                     # Show installed WOW! version
/wow --skip-image                  # Run pipeline skipping Image agent
/wow --skip-database               # Run pipeline skipping Database agent

Global Flags

FlagDescription
--dry-runAudit and plan only; make no changes
--domain=<url>Override the target domain for Lighthouse scans
--rollbackList restore points and revert to selected snapshot
--reportPrint the last run's HTML/JSON report path
--versionPrint installed WOW! version
--skip-<agent>Skip a named agent (e.g., --skip-image, --skip-seo)
--config=<path>Path to an alternate wow.config.json

Agent Reference

Each agent can be skipped individually using --skip-<agent-name>.

Agent Domain What it changes Skip flag
Orchestrator All Pipeline sequencing only — cannot be skipped —
Audit All Lighthouse scan, Core Web Vitals baseline, inventory --skip-audit
Backup Safety Full DB + files snapshot to wp-content/wow-backups/ --skip-backup (not recommended)
Caching Caching Server headers, browser cache rules, object cache config --skip-caching
Image Image Optimization Compress images, convert to WebP/AVIF, add lazy-load --skip-image
Security Security SSL enforcement, CSP/HSTS/X-Frame-Options headers, WP hardening --skip-security
SEO SEO Meta tags, Open Graph, JSON-LD schema, XML sitemap --skip-seo
CSS/JS Performance Minify, defer, and tree-shake CSS and JavaScript --skip-cssjs
Database Performance Query optimization, post revision cleanup, transient purge --skip-database
CDN Caching / Performance Push static assets to configured CDN provider --skip-cdn
Testing Safety Visual regression screenshots before/after execution --skip-testing (not recommended)

Configuration

WOW! looks for wow.config.json in the WordPress root directory. All fields are optional — WOW! uses sensible defaults when the file is absent.

{
  "domains": ["example.com"],
  "agents": {
    "image": {
      "quality": 85,
      "formats": ["webp", "avif"]
    },
    "caching": {
      "browser_ttl": 31536000
    },
    "cdn": {
      "provider": "cloudflare",
      "zone_id": "YOUR_ZONE_ID"
    }
  },
  "backup": {
    "offload": false,
    "offload_path": "/mnt/backups"
  },
  "report": {
    "format": "html",
    "output_dir": "wp-content/wow-reports"
  }
}

Configuration Fields

FieldTypeDefaultDescription
domainsstring[]auto-detectedTarget domains for Lighthouse scans
agents.image.qualitynumber82JPEG/WebP compression quality (1–100)
agents.image.formatsstring[]["webp"]Output formats (webp, avif)
backup.offloadbooleanfalseCopy backup to remote path after creation
report.formatstring"html"Report output format: html or json

WP-CLI Integration

WOW! is distributed as a WP-CLI package and integrates with the standard WP-CLI package manager.

  • Requires WP-CLI 2.5 or later — wp-cli.org
  • Installed globally via wp package install — available for all sites managed by WP-CLI on the server
  • Update with wp package update alo-labs/wow
  • Remove with wp package uninstall alo-labs/wow
  • Compatible with any WP-CLI setup (local, remote SSH, Trellis, Bedrock, etc.)

WOW! does not modify WordPress core files. All changes are reversible via /wow --rollback.
